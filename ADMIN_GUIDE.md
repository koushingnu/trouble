## 管理画面へのアクセス

### URL

```
https://your-domain.com/admin
```

### 必要な権限

- 管理者アカウント（`is_admin = true`）でログインしている必要があります
- 一般ユーザーはアクセスできません

### ログイン方法

1. トップページから「ログイン」をクリック
2. 管理者用のメールアドレスとパスワードを入力
3. ログイン後、ヘッダーメニューから「管理画面」にアクセス

## ユーザー管理

### ユーザー一覧の表示

管理画面の「ユーザー管理」タブで、すべてのユーザーを確認できます。

#### 表示項目

| 項目 | 説明 |
|------|------|
| ID | ユーザーID |
| メールアドレス | ログイン用メールアドレス |
| 名前 | ユーザーの姓名 |
| 電話番号 | 登録されている電話番号 |
| 認証キー | 紐付けられている認証キー |
| ステータス | 使用中/無効/未使用 |
| 管理者 | 管理者権限の有無 |
| 登録日 | アカウント作成日時 |
| 操作 | 編集・削除ボタン |

### ユーザーの検索

- メールアドレスで検索できます
- 部分一致検索に対応

### 管理者権限の付与・削除

1. 対象ユーザーの「編集」ボタンをクリック
2. 「管理者権限」のチェックボックスを変更
3. 「保存」をクリック

⚠️ **注意**: 管理者権限を付与すると、そのユーザーはすべての管理機能にアクセスできるようになります。

### ユーザーの削除

1. 対象ユーザーの「削除」ボタンをクリック
2. 確認ダイアログで「削除」を選択

⚠️ **注意**: 
- ユーザーを削除すると、そのユーザーの相談履歴もすべて削除されます
- 削除は取り消せません
- 削除されたユーザーのセッションは自動的に無効化されます

### CSV一括エクスポート

1. 「CSVダウンロード」ボタンをクリック
2. ユーザー一覧がCSV形式でダウンロードされます

**エクスポート項目:**
- ID
- メールアドレス
- 名前
- 電話番号
- 認証キー
- ステータス
- 管理者権限
- 登録日

## 認証キー管理

### 認証キーとは

- ユーザー登録時に必要な認証コード
- UUID形式（例: `4b98ef46-f837-11f0-a13d-9ca3ba2c942e`）
- 契約管理システムから発行されます

### 認証キー一覧の表示

管理画面の「認証キー管理」タブで、すべての認証キーを確認できます。

#### 表示項目

| 項目 | 説明 |
|------|------|
| ID | 認証キーID |
| 認証キー | UUID形式の認証コード |
| ステータス | 未使用/使用中/無効 |
| 使用者 | 紐付けられているユーザー |
| 作成日 | 認証キー生成日時 |
| 操作 | 編集・削除ボタン |

### ステータスの種類

| ステータス | 説明 | 表示色 |
|-----------|------|--------|
| 未使用 (UNUSED) | まだ誰も使用していない | グレー |
| 使用中 (ACTIVE) | ユーザーに紐付けられている | 緑 |
| 無効 (REVOKED) | 解約済み・使用不可 | 赤 |

### 新しい認証キーの生成

1. 「新規生成」ボタンをクリック
2. 生成数を入力（1〜100件）
3. 「生成」をクリック

生成された認証キーは、自動的にデータベースに登録されます。

### 認証キーのステータス変更

1. 対象認証キーの「編集」ボタンをクリック
2. ステータスを選択
3. 「保存」をクリック

### 認証キーの削除

1. 対象認証キーの「削除」ボタンをクリック
2. 確認ダイアログで「削除」を選択

⚠️ **注意**: 
- 使用中（ACTIVE）の認証キーは削除できません
- まずステータスを「無効」に変更してから削除してください

### CSV一括生成・ダウンロード

1. 「CSV生成」ボタンをクリック
2. 生成数を入力
3. 認証キーが生成され、CSV形式でダウンロードされます

**CSVフォーマット:**
```csv
認証キー,ステータス,作成日
4b98ef46-f837-11f0-a13d-9ca3ba2c942e,未使用,2026-02-19 10:00:00
...
```

## CSVインポート

### 概要

契約管理システムからエクスポートしたCSVファイルをアップロードして、認証キーとステータスを一括登録・更新できます。

### CSVフォーマット

#### 必須列

| 列番号 | 列名 | 説明 | 必須 |
|--------|------|------|------|
| 6 | ステータス | 承認/契約/解約/退会 | 任意 |
| 8 | 商品名 | 「トラブル解決ラボ」のみ抽出 | ✅ |
| 12 | 電話番号 | ハイフンなし10-11桁 | 任意 |
| 13 | 顧客ID | 顧客識別ID | ✅* |
| 15 | 認証キー | UUID形式 | ✅* |

*認証キーまたは顧客IDのどちらか一方が必須

#### サンプルCSV

```csv
"契約レコードSEQ","企業コード","法人名","個店コード","個店名","ステータス","決済","商品名","料金","ユ−ザ−ID","管理ID","電話番号","顧客ID","トラッキングID","認証キー","契約日","課金日","解約日","解約種別","解約理由","解約理由(その他)","再請求結果","予備１","予備２","予備３","予備４"
"999999","0623","テスト会社","0800","テストショップ","承認","クレジットカード","トラブル解決ラボ","660","TEST001","","07012345678","test-customer-001","","4b98ef46-f837-11f0-a13d-9ca3ba2c942e","2026-01-29","2026-03-01","","","","","","","","",""
```

### インポート手順

#### 1. CSVファイルを準備

- 契約管理システムからCSVをエクスポート
- Shift-JISまたはUTF-8エンコーディングに対応

#### 2. CSVをアップロード

1. 管理画面の「CSVインポート」タブを選択
2. 「CSVファイルを選択」ボタンをクリック、またはファイルをドラッグ&ドロップ
3. 「アップロード」ボタンをクリック

#### 3. プレビュー確認

アップロード後、以下の情報が表示されます：

- **対象レコード数**: 「トラブル解決ラボ」のレコード数
- **スキップされたレコード数**: 認証キーが空のレコード数
- **抽出データ一覧**: 各レコードの詳細

**プレビュー項目:**
- 行番号
- 認証キー
- 電話番号
- ステータス（承認/契約/解約）
- マッピング後のステータス（使用中/無効/未使用）
- 顧客名（DBから取得）
- 判定（対象/スキップ）

#### 4. 内容を確認

プレビュー画面で以下を確認してください：

- ✅ 認証キーが正しいか
- ✅ 電話番号の形式が正しいか（ハイフンなし）
- ✅ ステータスが正しくマッピングされているか
- ✅ スキップされたレコードの理由を確認

#### 5. DBに登録

内容に問題がなければ、「DBに登録」ボタンをクリックします。

⚠️ **注意**: 
- 登録後の取り消しはできません
- 必ずプレビューで内容を確認してから登録してください

#### 6. 登録結果の確認

登録完了後、以下の統計が表示されます：

| 項目 | 説明 |
|------|------|
| 処理数 | 処理したレコード数 |
| 成功数 | 正常に登録・更新されたレコード数 |
| 新規作成数 | 新しく作成された認証キー数 |
| 更新数 | ステータスが更新された認証キー数 |
| スキップ数 | 変更がなくスキップされたレコード数 |
| 失敗数 | エラーが発生したレコード数 |
| 電話番号更新数 | 電話番号が更新されたユーザー数 |
| エラー詳細 | 失敗したレコードの詳細 |

### ステータスマッピング

| CSVの値 | DBのステータス | 説明 |
|---------|---------------|------|
| `承認` または `契約` | `ACTIVE` | 使用中 |
| `退会` または `解約` | `REVOKED` | 無効 |
| その他 | `UNUSED` | 未使用 |

### 処理ロジック

#### 認証キー（Token）の処理

1. **既存の認証キーの場合**
   - ステータスを更新
   - ステータスが変更された場合のみ「更新」としてカウント
   - ステータスが同じ場合は「スキップ」としてカウント

2. **新規の認証キーの場合**
   - 新しい認証キーを作成
   - 「新規作成」としてカウント

#### 電話番号の更新

- 該当トークンに紐づくユーザーが存在する
- **かつ、ユーザーの電話番号が未設定の場合のみ**更新
- 既に電話番号が設定されているユーザーは上書きされません

#### 解約（REVOKED）の扱い

- トークンのステータスを「無効」に変更
- **ユーザーアカウントは削除されません**
- ログイン時にトークンの有効性チェックで弾かれます

#### エラーハンドリング

- 各レコードを個別に処理
- エラーが発生しても他のレコードは処理を継続
- 成功したレコードだけコミット（部分的成功）

### 制限事項

#### Vercel制限

| 項目 | Hobby | Pro |
|------|-------|-----|
| リクエストボディサイズ | 4.5MB | 4.5MB |
| 関数実行時間 | 10秒 | 60秒 |
| 推奨レコード数 | ~500件 | ~2000件 |

#### 推奨事項

- ✅ 大量データの場合: CSVを分割してアップロード
- ✅ 定期実行の場合: 差分データのみをアップロード
- ✅ バックアップ: インポート前にDBバックアップを推奨

### トラブルシューティング

#### エラー: 「ファイルが選択されていません」
→ CSVファイルを選択してからアップロードしてください

#### エラー: 「管理者権限が必要です」
→ 管理者アカウントでログインしてください

#### エラー: 「CSVインポート中にエラーが発生しました」
→ CSVのフォーマットを確認してください（ヘッダー行、文字エンコーディングなど）

#### スキップが多い場合
→ 「認証キー」列が空の行が多い可能性があります。詳細ログを確認してください

### バックアップ

#### 自動バックアップ

- AWS Lightsail Databaseは自動バックアップ機能あり
- 推奨設定: 日次自動バックアップ、7日間保持

#### 手動バックアップ

重要な操作（大量データのインポート、スキーマ変更など）の前に、手動でバックアップを取得してください。

```bash
# MySQLダンプを取得
mysqldump -h [HOST] -u [USER] -p [DATABASE] > backup_$(date +%Y%m%d).sql
```

### データのアーカイブ

#### 古いチャットデータのアーカイブ（推奨）

6ヶ月以上前の解決済みチャットをアーカイブすることで、データベース容量を70-80%削減できます。

**アーカイブ対象:**
- ステータスが「解決済み」
- 最終更新日が6ヶ月以上前

**アーカイブ方法:**
1. 対象データをS3にエクスポート
2. データベースから削除
3. 必要に応じて復元

## 監視とメンテナンス

### 監視すべきメトリクス

| メトリクス | 正常範囲 | 警告 | 危険 | 対応策 |
|-----------|---------|------|------|--------|
| CPU使用率 | 〜50% | 50-80% | 80%以上 | インスタンスサイズアップ |
| メモリ使用率 | 〜60% | 60-85% | 85%以上 | クエリ最適化/サイズアップ |
| ストレージ使用率 | 〜50% | 50-70% | 70%以上 | ストレージ拡張/アーカイブ |
| 接続数 | 〜50% | 50-80% | 80%以上 | コネクションプール見直し |

### 定期メンテナンス

#### 毎日
- ✅ エラーログの確認
- ✅ ユーザー登録数の確認
- ✅ チャット利用状況の確認

#### 毎週
- ✅ データベース容量の確認
- ✅ パフォーマンスメトリクスの確認
- ✅ バックアップの確認

#### 毎月
- ✅ ユーザー統計レポートの作成

- ✅ セキュリティアップデートの適用

### スケーリング計画

#### 短期（現在〜500ユーザー）
- ✅ 現状維持で問題なし

## セキュリティ

### アクセス制御

- ✅ 管理画面は管理者権限（`is_admin = true`）が必要
- ✅ セッション認証（NextAuth.js）
- ✅ ミドルウェアによるルート保護

### データ保護

- ✅ SSL接続必須
- ✅ パスワードはbcryptでハッシュ化
- ✅ 環境変数で機密情報を管理

### 監査ログ

現在、以下の操作がログに記録されます：
- ユーザーログイン
- ユーザー登録
- ユーザー削除
- 認証キーの生成・削除
- CSVインポート
