# CSVインポート機能 ガイド

## 📋 概要

管理画面から契約情報CSVをアップロードして、認証キーとステータスを一括登録・更新できる機能です。

---

## 🎯 機能

### 1. **CSVファイルのアップロード**
- Shift-JISまたはUTF-8エンコーディングに対応
- ファイルサイズ制限: 最大4.5MB（Vercel制限）
- 処理時間制限: 最大60秒（Vercel Pro）

### 2. **データ抽出**
以下のフィールドを抽出してDBに登録します：

| フィールド名 | 列位置 | 必須 | 説明 |
|------------|--------|------|------|
| **認証キー** | 15列目 | ✅ 必須* | UUID形式の認証キー（`token_value`） |
| **顧客ID** | 13列目 | ✅ 必須* | 顧客ID（認証キーが空の場合に使用） |
| **電話番号** | 12列目 | 任意 | ハイフンなし10-11桁の電話番号 |
| **ステータス** | 6列目 | 任意 | 契約状態（承認/契約/退会/解約など） |

*認証キーまたは顧客IDのどちらか一方が必須です。

### 3. **ステータスマッピング**

| CSVの値 | DBのステータス | 説明 |
|---------|---------------|------|
| `承認` または `契約` | `ACTIVE` | 使用中 |
| `退会` または `解約` | `REVOKED` | 無効 |
| その他 | `UNUSED` | 未使用 |

### 4. **処理ロジック**

#### プレビューフェーズ（自動）
1. CSVファイルをアップロード
2. 「トラブル解決ラボ」のレコードを抽出
3. プレビュー画面に表示

#### 登録フェーズ（手動確認後）
1. **認証キー（Token）の処理**
   - 既存の場合 → ステータスを更新
   - 新規の場合 → 新しい認証キーを作成

2. **電話番号の更新**
   - 該当トークンに紐づくユーザーが存在
   - かつ、ユーザーの電話番号が未設定の場合のみ更新

3. **解約（REVOKED）の扱い**
   - トークンのステータスを「無効」に変更
   - ユーザーはそのまま残す（ログイン時にトークンチェックで弾かれる）

4. **エラーハンドリング**
   - 各レコードを個別に処理
   - エラーが発生しても他のレコードは処理を継続
   - 成功したレコードだけコミット（部分的成功）

### 重要な列

| 列番号 | 列名 | 説明 |
|--------|------|------|
| 6 | ステータス | 承認/契約/解約/退会 |
| 12 | 電話番号 | ユーザーの電話番号 |
| 13 | 顧客ID | 顧客識別ID（認証キーの代替） |

### 注意事項

- ✅ ヘッダー行（1行目）は必須です
- ✅ 認証キー（15列目）または顧客ID（13列目）のどちらかが必須
- ✅ 認証キーと顧客IDが両方とも空の行はスキップされます
- ✅ 電話番号は任意（空欄可）
- ✅ ステータスは任意（空欄の場合は「未使用」）
- ⚠️ 認証キーの重複は更新として処理されます
- 📝 認証キーが空の場合、顧客IDを認証キーとして使用します

## 🚀 使い方

### 1. 管理画面にアクセス

```
https://your-domain.com/admin
```

### 2. 「CSVインポート」タブを選択

管理画面の3つ目のタブ「CSVインポート」をクリックします。

### 3. CSVファイルを選択

「CSVファイルを選択」ボタンまたはドラッグ&ドロップで、アップロードするCSVファイルを選択します。

### 4. プレビュー表示

「アップロード」ボタンをクリックすると、抽出結果がプレビュー表示されます。
- 対象レコード数（トラブル解決ラボ）
- スキップされたレコード数
- 各レコードの詳細

### 5. 内容を確認

プレビュー画面で以下を確認します：
- 認証キー
- 電話番号
- ステータス（使用中/無効/未使用）
- 判定（対象/スキップ）

### 6. DBに登録

内容に問題がなければ、「DBに登録」ボタンをクリックして、データベースに反映します。

### 7. 登録結果の確認

登録完了後、以下の統計が表示されます：
- 処理数
- 成功数
- 新規作成数
- 更新数
- 失敗数
- エラー詳細（失敗した場合）

## ⚠️ 重要な注意事項

### 電話番号の上書きルール
- **未設定の場合のみ**更新されます
- 既に電話番号が設定されているユーザーは上書きされません
- これにより、ユーザーが自分で変更した電話番号を保護します

### 解約（REVOKED）トークンの扱い
- トークンのステータスを「無効」に変更
- **ユーザーアカウントは削除されません**
- ログイン時にトークンの有効性チェックで弾かれます

### エラーハンドリング
- **部分的成功を許容**します
- 一部のレコードでエラーが発生しても、他のレコードは正常に処理されます
- エラーの詳細は登録結果画面で確認できます

### データの安全性
1. **2段階確認**
   - まずプレビューで内容を確認
   - 確認後、手動で「DBに登録」ボタンをクリック

2. **ロールバックなし**
   - 登録後の取り消しはできません
   - 必ずプレビューで内容を確認してから登録してください

3. **バックアップ推奨**
   - 大量のデータを登録する前に、DBのバックアップを推奨します

## 🔧 技術仕様

### API エンドポイント

#### 1. プレビューAPI
```
POST /api/admin/import-csv
```

**リクエスト:**
```typescript
FormData {
  file: File (CSV)
}
```

**レスポンス:**
```typescript
{
  success: boolean;
  message: string;
  stats: {
    total: number;
    filtered: number;
    skipped: number;
  };
  extractedData: Array<{
    rowNumber: number;
    authKey: string;
    phoneNumber: string;
    status: string;
    statusMapped: "ACTIVE" | "REVOKED" | "UNUSED";
    isFiltered: boolean;
    skipReason?: string;
  }>;
}
```

#### 2. 登録確認API
```
POST /api/admin/import-csv/confirm
```

**リクエスト:**
```typescript
{
  records: Array<{
    authKey: string;
    phoneNumber: string;
    status: "ACTIVE" | "REVOKED" | "UNUSED";
  }>;
}
```

**レスポンス:**
```typescript
{
  success: boolean;
  message: string;
  results: {
    total: number;
    success: number;
    failed: number;
    created: number;
    updated: number;
    phoneUpdated: number;
    errors: Array<{
      authKey: string;
      error: string;
    }>;
  };
}
```

### 使用ライブラリ

- `csv-parse`: CSV解析
- `iconv-lite`: 文字エンコーディング変換（Shift-JIS対応）

## ⚠️ 制限事項

### Vercel制限

| 項目 | Hobby | Pro |
|------|-------|-----|
| リクエストボディサイズ | 4.5MB | 4.5MB |
| 関数実行時間 | 10秒 | 60秒 |
| 推奨レコード数 | ~500件 | ~2000件 |

### 推奨事項

- ✅ **大量データの場合**: CSVを分割してアップロード
- ✅ **定期実行の場合**: 差分データのみをアップロード
- ✅ **バックアップ**: インポート前にDBバックアップを推奨

---

## 🐛 トラブルシューティング

### エラー: 「ファイルが選択されていません」
→ CSVファイルを選択してからアップロードしてください

### エラー: 「管理者権限が必要です」
→ 管理者アカウントでログインしてください

### エラー: 「CSVインポート中にエラーが発生しました」
→ CSVのフォーマットを確認してください（ヘッダー行、文字エンコーディングなど）

### スキップが多い場合
→ 「認証キー」列が空の行が多い可能性があります。詳細ログを確認してください
